# Quickstart: 日报助手 MVP 验证说明

## 1. 目标

用于开发完成后的最小人工验收，验证 MVP 核心闭环在 Windows 11 上可运行、可降级、可复制。

## 2. 预置条件

- 应用已成功安装或以开发模式启动
- 本地数据库已初始化
- 默认快捷键与默认提醒配置可用
- 如需验证 AI 润色，需先配置一组可用的 OpenAI 风格服务参数

## 3. 手动验收步骤

### 场景 A：快速记录闭环

1. 启动应用，确认应用进入后台常驻且托盘入口存在。
2. 按下全局快捷键，确认快速记录窗口弹出并自动聚焦输入框。
3. 输入“整理日报助手 MVP 计划”并保存。
4. 再次按下快捷键，输入“确认 AI 降级策略”并保存。
5. 打开当日汇总窗口，确认能看到两条任务。

### 场景 B：汇总编辑与基础日报生成

1. 在汇总窗口中将第一条任务保留为“已完成”。
2. 将第二条任务改为“进行中”，并填写备注“等待接口联调确认”。
3. 点击“生成基础日报”。
4. 确认结果文本中包含“已完成”和“进行中”两个分组，并反映备注变更。

### 场景 C：AI 润色成功

1. 打开设置页，填写有效的 Base URL、API Key 与模型名并保存。
2. 返回汇总窗口，点击“AI 润色”。
3. 确认生成结果比基础日报更自然，但未夸大原始任务内容。

### 场景 D：AI 润色失败降级

1. 将模型名改为无效值或断开网络。
2. 再次点击“AI 润色”。
3. 确认界面提示 AI 失败，但基础日报结果仍保留且可复制。

### 场景 E：复制结果

1. 点击“复制结果”。
2. 在任意文本输入位置粘贴，确认内容与结果区一致。
3. 若模拟复制失败，确认界面给出失败提示且文本未丢失。

### 场景 F：提醒触发

1. 将下班时间调整为当前时间后 6 分钟。
2. 等待 1 分钟后，确认汇总窗口在“下班前 5 分钟”时自动弹出。
3. 修改下班时间，再次确认后续提醒按新设置生效。

## 4. 回归重点

- 快捷键修改后重新注册是否成功
- 当天无任务时打开汇总窗口的空状态
- 备注修改后再次生成是否总是使用最新内容
- 应用重启后任务和设置是否仍可读取

## 5. 通过标准

- 核心闭环全程可在本地完成
- AI 失败不会阻塞基础日报
- 数据不会因异常操作被覆盖或丢失
- 剪贴板复制有明确成功/失败反馈
