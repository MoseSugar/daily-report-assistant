# AI Polish Contract: 日报助手 MVP

## 1. 目标

定义与兼容 OpenAI 风格 API 的 AI 服务交互约束，保证可以在不修改业务层的情况下替换具体服务。

## 2. 输入契约

### 配置输入

- `base_url`: 用户配置的服务地址
- `api_key`: 用户配置的访问凭证
- `model`: 用户配置的模型名
- `timeout_seconds`: 请求超时时间，MVP 可采用固定默认值

### 业务输入

- `date`: 当日日期
- `done_entries`: 已完成任务列表
- `in_progress_entries`: 进行中任务列表
- `notes`: 任务备注集合
- `basic_report_text`: 已生成的基础日报文本

## 3. Prompt 约束

AI 请求必须明确表达以下约束：

- 输出为一段适合直接复制发送的工作日报文本。
- 语言自然、简洁、专业。
- 明确区分已完成与进行中。
- 尽量保留用户原始任务信息，不夸大事实。
- 若原始内容较少，应保持克制，不补充用户未提供的成果。

## 4. 成功响应契约

- 从 AI 响应中提取纯文本日报内容。
- 返回结果必须非空。
- 返回文本必须可直接展示和复制。

## 5. 失败响应契约

以下情况统一视为失败：

- 配置缺失
- 网络错误
- 超时
- 非成功状态码
- 返回结构无法解析
- 解析后内容为空

失败时必须返回：

- `error_code`
- `message`
- `fallback_to_basic`: true

## 6. 安全约束

- `api_key` 不得写入日志、错误提示或调试输出。
- 请求失败日志不得包含完整请求体中的敏感信息。
- 当用户禁用 AI 时，不得发起任何网络调用。
